# 从零开始搭建 Chirpy 主题个人博客指南

本文汇总了使用 GitHub Pages + Chirpy 主题搭建个人技术博客的完整流程、关键配置及常见问题排查。

## 一、 环境准备与仓库创建

1.  **创建仓库**：访问 [chirpy-starter](https://github.com/cotes2020/chirpy-starter) 模板仓库，点击 **Use this template**。
2.  **命名规范**：仓库名务必设为 `你的用户名.github.io`（例如：`Pure-yichu.github.io`）。
3.  **权限开启（核心）**：
    * 进入仓库 **Settings > Actions > General**。
    * 在 **Workflow permissions** 中勾选 **Read and write permissions** 并保存。
4.  **发布源设置**：
    * 进入 **Settings > Pages**。
    * 将 **Build and deployment > Source** 切换为 **GitHub Actions**。

## 二、 本地环境配置

1.  **克隆仓库**（如遇网络问题可使用镜像）：
    ```powershell
    git clone [https://github.com/Pure-yichu/Pure-yichu.github.io.git](https://github.com/Pure-yichu/Pure-yichu.github.io.git)
    cd Pure-yichu.github.io
    ```
2.  **修改基础配置 (`_config.yml`)**：
    * `title`: 博客名称。
    * `tagline`: 博客副标题。
    * `url`: 'https://pure-yichu.github.io'（末尾不要加斜杠）。
    * `baseurl`: ""（必须为空字符串）。
    * `timezone`: Asia/Shanghai（解决文章因时差被误判为“未来发布”而不显示的问题）。
    * `avatar`: '/assets/img/avatar.jpg'（头像路径，建议存放在 assets/img/ 文件夹）。

## 三、 文章发布规范

所有文章必须存放在 `_posts/` 目录下。

1.  **文件名格式**：`YYYY-MM-DD-文件名.md`
    * 例如：`2026-01-17-python-and-docker-study.md`
2.  **文档头（YAML Front Matter）示例**：
    ```markdown
    ---
    title: Python 与 Docker 学习笔记  # 你的兴趣方向
    date: 2026-01-17 20:00:00 +0800
    categories: [技术, 学习记录]
    tags: [Python, Docker]
    layout: post
    ---
    ```
3.  **大纲功能**：只要在正文中使用 `##` 或 `###` 标题，Chirpy 会自动在右侧生成悬浮大纲。

## 四、 Git 部署三步曲

每当修改完文件或新增文章后，运行以下命令：
```powershell
git add .
git commit -m "备注你的修改内容"
git push origin main  # 确保分支名为 main
```




## 五、 本地预览 (使用 Docker)
如果你对 Docker 感兴趣，可以使用以下命令进行本地实时预览，无需配置 Ruby 环境：

```PowerShell

docker run --rm -it `
  -v "${PWD}:/srv/jekyll" `
  -p 4000:4000 `
  jekyll/jekyll:latest `
  jekyll serve --watch --incremental --force_polling
```
运行后访问 localhost:4000 即可预览。如果文章未显示，终端日志会提示具体的错误原因。

## 六、 常见问题排查 (Checklist)
404 错误：检查 _config.yml 里的 baseurl 是否为空。

文章不显示：检查文件名日期是否正确，以及 _config.yml 里的时区设置。

Actions 不运行：检查 .github/workflows/pages-deploy.yml 里的分支名称是否与你 GitHub 上的分支（main/master）一致。

样式乱码：检查 GitHub Pages 的 Source 是否已切换为 GitHub Actions。
